## 규칙

1. 디스코드 스크럼 전까지 **각 주차에 해당하는 개념별 문제들을 풀이, 풀이 과정 설명을 준비**합니다.
    - 너무 자세하게 하지 않으셔도 괜찮아요 :)
    
2. 디스코드 스크럼 시간에 5~10분 분량으로 **각자 준비한 문제별 풀이 또는 코드 리뷰를 발표**합니다. 
    - 새롭게 알게 된 내용, 중요하다고 생각되는 내용, 새로 생긴 궁금증, 활용 사례 등등…
    
3. 그 주에 발표할 문제별 풀이는 **다음주 수요일 스터디 시작 전**까지 과제 제출란에 업로드합니다.
   
4. **프로그래머스**에서 주제에 따른 문제를 선택하여 문제를 풀고 **깃허브 알고리즘 레포지토리**에 **자신의 풀이 PR**을 남깁니다.
